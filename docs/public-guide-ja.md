# Dua 公開ガイド（日本語）

このドキュメントは、Dua を外部公開・配布するときの最小ガイドです。

## 1. プロジェクトの目的を明確にする

Dua は「汎用 VM」ではなく、**D アプリへ組み込むスクリプトレイヤー**を目的としています。  
README や公開ページでは、次の点を先に示すと伝わりやすくなります。

- 何を解決するか（設定・ルール・拡張ロジックの外出し）
- どこまで対応するか（軽量重視、互換目的ではない）
- どの API から触ればよいか（`Dua.ScriptEngine` / `Dua.Value`）

## 2. 初学者向けの導線を固定する

最低限、以下 3 つを README に置いてください。

1. ビルド手順（`dub build --compiler=ldc2`）
2. 実行手順（`dub run --compiler=ldc2`）
3. 10〜20 行の最小埋め込み例

## 3. API 公開方針

外部公開時は `module dua`（`source/dua/package.d`）を入口にします。  
内部実装（`runtime.d`, `parser.d` など）への直接依存は推奨しません。

### 推奨 API

- `Dua.ScriptEngine`
- `Dua.Value`
- `Dua.RunOutcome`

## 5. セキュリティと運用上の注意

スクリプトを外部入力として扱う場合、運用側で以下を検討してください。

- スクリプト実行対象の制限（読み込み元や保存先）
- 公開するネイティブ関数の最小化（`bindNative` の絞り込み）
- タイムアウト・リソース制限（必要に応じたホスト側制御）
